# Epistola

Современный мессенджер на Flutter и Firebase с личными чатами, группами, ролями, системой модерации и защитой через Firestore Rules.

> Текущая версия разработки: **v0.5.7**
>
> Статус проекта: **Активная разработка**

---

# О проекте

**Epistola** — это собственный мессенджер, разрабатываемый на Flutter с использованием Firebase.

Проект начинался как учебный, но постепенно вырос в полноценную систему общения с поддержкой:

* личных чатов;
* групповых чатов;
* пользовательских профилей;
* ролей и прав доступа;
* модерации;
* Firestore Security Rules;
* постоянной авторизации пользователей.

Долгосрочная цель проекта — создать удобную платформу для общения рабочих коллективов, организаций и команд.

# Реализованные возможности

## Авторизация

* Регистрация пользователей
* Вход в аккаунт
* Выход из аккаунта
* Автоматический вход после перезапуска приложения
* Запоминание последнего использованного email

---

## Профиль пользователя

Поддерживаются:

* имя пользователя;
* email;
* номер телефона;
* информация «О себе».

---

## Личные чаты

* создание личных переписок;
* стабильный идентификатор чата между двумя пользователями;
* отправка сообщений в реальном времени;
* сохранение истории сообщений.

---

## Групповые чаты

* создание групп;
* добавление участников;
* просмотр информации о группе;
* управление участниками.

---

## Система ролей

Поддерживаемые роли:

* Владелец (Owner)
* Администратор (Admin)
* Модератор (Moderator)
* Участник (Member)
* Гость (Guest)

---

## Система модерации

Поддерживаются:

* Мьют
* Размьют
* Бан
* Разбан

Дополнительно:

* временные ограничения;
* постоянные ограничения;
* автоматическое снятие ограничений по времени;
* хранение причины наказания.

---

## Настройки группы

Права на отправку сообщений:

* Все участники
* Только модераторы и выше
* Только администраторы

---

## Безопасность

Реализована защита через Firestore Rules:

* доступ к чатам только участникам;
* доступ к сообщениям только участникам;
* защита профилей пользователей;
* контроль ролей;
* контроль модерации;
* защита последнего администратора группы;
* защита личных чатов.

## Рефакторинг интерфейса

Крупные экраны постепенно разбиваются на независимые виджеты.

Пример:

```text
widgets/chat/

├── chat_app_bar.dart
├── banned_overlay.dart
└── message_input_area.dart
```

```text
widgets/group/

├── group_header.dart
├── group_members_section.dart
├── member_role_card.dart
├── moderation_actions_section.dart
├── role_selection_sheet.dart
└── ...
```

Это позволяет поддерживать проект в чистом и понятном состоянии по мере роста функционала.

# Принципы разработки

Проект придерживается следующих правил:

* UI отделяется от бизнес-логики;
* сервисы не содержат интерфейс;
* переиспользуемые элементы выносятся в отдельные виджеты;
* большие файлы постепенно разделяются на логические блоки;
* критически важные операции дополнительно защищаются Firestore Rules.
