## ⚡ О проекте

**Epistola** — современный корпоративный мессенджер, разрабатываемый на **Flutter** с использованием **Firebase**.

Проект создаётся как единая коммуникационная платформа для сотрудников компании. Основная цель — объединить привычное удобство современных мессенджеров с возможностями корпоративной рабочей среды.

В долгосрочной перспективе Epistola станет не только мессенджером, но и платформой с модульными рабочими пространствами (**Spaces**), мини-приложениями и инструментами для совместной работы.

# 🏗 Архитектура

В версии **v0.5.10** выполнен полный рефакторинг сервисов.

Монолитный `ChatService` был разделён на специализированные сервисы при полном сохранении внешнего API.

```text
UI (Screens)
        │
        ▼
ChatService (Facade)
        │
 ┌──────┼────────────────────────────────────┐
 ▼      ▼        ▼        ▼        ▼        ▼
Messages Groups Members Permissions Private Search
        │
        ▼
Firebase
        │
        ▼
Cloud Firestore
```

### Структура сервисов

```text
lib/services/chat/

chat_base_service.dart
chat_messages_service.dart
chat_private_service.dart
chat_search_service.dart
chat_members_service.dart
chat_groups_service.dart
chat_permissions_service.dart
```

Такое разделение упрощает сопровождение проекта и подготовило архитектуру к следующим крупным этапам разработки.

# 🧩 Spaces

Spaces — стратегическое направление развития Epistola.

Spaces не рассматриваются как обычные групповые чаты. Это будущая платформа рабочих мини-приложений.

Планируемые модули:

- график смен;
- рабочие задачи;
- объявления;
- документы;
- журналы;
- заявки;
- рабочие панели.

Администраторы смогут определять список доступных модулей для пространства, а каждый пользователь сможет включать или скрывать модули для себя.

# 🎯 Принципы разработки

Проект развивается по следующим принципам:

- Архитектура важнее скорости разработки.
- Сначала проектирование — потом реализация.
- Минималистичный и понятный интерфейс.
- Material 3.
- UX с вдохновением от Telegram, но со своей идентичностью.
- После каждого крупного этапа обновляется документация.
- После каждой стабильной версии создаётся Git Tag.
- Код должен проходить `flutter analyze` без ошибок и предупреждений.
- Для Firebase CLI на Windows используются команды через `firebase.cmd`.
